# 工程机械行业研究 Benchmark Question v0.5

## 任务

你是一名覆盖机械设备与先进制造的行业分析师。请站在 **2024 年 10 月前后** 的信息时点，围绕以下主题完成一份 **行业深度研究报告**：

`工程机械行业：国内需求是否走向成熟筑底，海外市场如何分层演绎，哪些品类、区域与公司特征最受益？`

你的任务不是写单一公司的首次覆盖，也不是写泛泛的行业介绍，而是交付一份 **像优秀人类行业深度研报一样** 的研究报告：它应当有主线、有比较、有取舍、有深挖，也有可追溯的审计附录。

## 资料边界

1. 你可以搜索和使用公开资料。
2. 你必须披露所有实际使用的资料来源。
3. 关键判断、数字、日期、份额、增速、区域结论、公司经营事实，都必须可追溯。
4. 如果不同来源存在冲突，你可以选择其一，但必须说明采信逻辑。
5. 如果你的结论依赖推断或假设，必须明确标注。
6. 你的分析应尽量站在 **2024 年 10 月前后** 可获得的信息基础上展开；如果使用更晚资料，只能作为补充佐证，不能偷换题目时点下的主判断。

## 先想清楚，再成稿

在开始写正文之前，请你先在脑中完成以下工作，但**不要输出中间思维过程**：

1. 识别你认为这份行业报告真正的主线是什么。
2. 选出决定行业判断的 **1-2 个最重要结构变量**。
3. 识别一个最强的反方观点，判断它为什么不足以推翻你的主线。
4. 想清楚“为什么是现在”，而不是停留在长期正确但当前无交易意义的泛判断。
5. 想清楚行业变化如何传导到品类、区域与公司特征，而不是只点公司名。

最终你只输出报告本身，不输出思维草稿。

## 你要回答的核心问题

你的报告至少要回答下面七个问题：

1. 未来 1-3 年，工程机械行业最重要的结构性变化是什么？
2. 你的主线最可能被什么反方观点挑战？你为什么仍然选择当前判断？
3. 国内市场应按哪些需求来源和品类来拆解，为什么？
4. 海外市场应按哪些区域层次来拆解，为什么？
5. 哪些品类更可能先改善，哪些品类仍然偏弱？
6. 什么样的公司或能力更受益，映射逻辑是什么？
7. 哪些变量会证伪你的行业主线？

## 写作导向

请把正文写成 **给人看的行业研究报告**，而不是考试题答案。

你应当追求的质感是：

- 有清楚的主线，不平均用力
- 有行业拆解框架，不是材料堆砌
- 有显式比较，不是并列罗列
- 有“为什么重要”的解释，不只列事实
- 有主线与反方的取舍，而不是单向看多
- 有受益方向映射，不停在行业综述

你可以使用小标题、表格和少量 bullet，但正文不应主要依赖 bullet 推进。

正文建议达到 **3000-5000 字中文量级**。这不是硬性字数门槛，但如果篇幅过短且没有展开，通常意味着你没有完成真正的行业研究。

正文至少一半篇幅应围绕 **1-2 个核心结构变量** 展开，而不是把每个点平均写一遍。

至少有一个关键分析块应被连续多段展开，并同时回答：

- 这件事为什么重要
- 它如何传导到品类、区域或公司特征
- 为什么相反观点不足以推翻你的主线

对“关键争议与主线取舍”“国内市场”“海外市场”“结构性机会与受益方向”这些关键部分，建议先给本节判断，再展开论证和数据支持。

附录服务于**审计与可追溯性**，不应替代正文分析。

## 各部分要求

### 核心判断

- 用自然段写出你的行业主线
- 明确指出未来 1-3 年最重要的 1-2 个结构变量
- 应让读者在报告开头就知道：你认为这个行业最大的变化是什么

### 关键争议与主线取舍

- 说明你认为最强的反方观点是什么
- 说明它为什么不足以推翻你的主线
- 最好给出事实、数字或可验证依据，而不是抽象反驳

### 行业拆解框架

- 说明你按什么维度拆这个行业
- 这些维度必须真正解释行业变化，而不是机械罗列
- 国内与海外的拆法可以不同，但都要有解释力

### 国内市场：需求与品类分层

- 至少覆盖两个需求来源维度，例如但不限于：
  - 地产
  - 基建
  - 采矿
  - 更新替换
- 至少覆盖两个品类维度，例如但不限于：
  - 挖掘机
  - 起重机
  - 混凝土机械
  - 高机
  - 矿机
- 需要说明哪些环节先改善、哪些环节仍弱，以及原因

### 海外市场：区域分层与竞争机会

- 至少对两个区域层次进行横向比较，例如但不限于：
  - 高端成熟市场
  - 机会市场
  - 稳增长市场
- 需要解释不同区域的空间、增速、竞争格局和进入壁垒差异
- 不能只说“海外空间大”，必须做分层

### 结构性机会与受益方向

- 不要求给唯一推荐股票
- 但必须回答“什么样的公司/能力更受益”
- 受益方向应优先写成：
  - 公司特征
  - 品类暴露
  - 区域布局
  - 海外渠道能力
  - 后市场/服务能力
  - 产品结构优势
- 你可以点公司名，但公司名应当只是例子，而不是替代受益逻辑本身

### 风险与证伪

- 至少写出 2 个会削弱你主线的风险
- 至少写出 2 条证伪条件
- 证伪条件要尽量可观察、可验证
- 风险不应只是 checklist，而应说明它削弱的是哪一段主线

## 引用与审计要求

1. 正文中的关键判断、数字、日期、第三方事实、管理层口径、份额/增速/销量结论，必须在句末带来源标注，如 `[R01][R04]`。
2. Source Register 中的 Source ID 必须与正文和附录内的引用一致。
3. Key Claim Map 中的每条 claim 必须标注 `fact / inference / assumption`，并给出 supporting source ids。
4. 如果某个判断依赖你自己的推断，正文与 Assumption List 都应显式承认这一点。

### Source Register

对每个实际使用的来源至少记录以下字段：

- `Source ID`
- `标题`
- `来源类型`
- `URL`
- `来源等级`
- `source_depth`
- `used_for`

来源等级请使用：

- `A` 官方披露 / 官方文件 / 官方网站
- `B` 专业终端 / 数据库 / 行业数据库
- `C` 机构研报 / 研究机构
- `D` 主流媒体 / 行业媒体
- `E` 社交平台 / 社区讨论
- `F` 未验证小道消息

其中 `source_depth` 只需使用：

- `thin`
- `medium`
- `thick`

### Key Claim Map

列出最重要的 5-10 条判断，并标注：

- `claim`
- `fact / inference / assumption`
- `supporting source ids`

### Core Numbers Table

列出正文真正使用过的关键数字，并标注：

- 指标
- 时间口径
- 数值
- source id

### Assumption List

列出你自行引入的关键假设，并说明：

- 这个假设影响哪个判断
- 如果假设不成立，会影响什么

## 评分提醒

这道题高分不是靠“覆盖很多点”，而是靠：

- 主线清楚
- 国内/海外拆解合理
- 品类与区域比较成立
- 主线与反方取舍成立
- 受益映射有投资意义
- 来源治理过关

低分常见原因包括：

- 写成工程机械行业综述
- 平均覆盖、没有主次
- 只有事实没有解释
- 只说行业，不做受益映射
- 海外部分只写“空间大”，没有区域分层
- 点公司名但不解释受益逻辑
- 来源披露不完整
